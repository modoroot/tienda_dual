API: guía de uso

Crear solicitudes

Primero necesitarás un token válido. Este token te lo debe proporcionar un administrador o usuario ya registrado

Ejemplo

- Eliminar un usuario { "id_usuario": 1 "token": "<\inserta-token\>" }

En caso de pedir datos, podrás parametrizar con "page" o "id". Te devolverá un JSON en función de lo solicitado

          - Ejemplo: http://localhost/tienda_dual/tienda_dual/api_rest/usuarios?id=30
          [
            {
              "id_usuario": "30",
              "nombre": "",
              "username": "usuario",
              "password": "b665e217b51994789b02b1838e730d6b93baa30f",
              "email": "",
              "id_privilegio": "64"
            }
          ]

Referencias

Ruta final Método Parámetros Descripción
/usuarios GET Te devuelve los usuarios
/usuarios GET page=? Te devuelve los usuarios según el número de página indicado
/usuarios GET id=? Te devuelve el usuario según el ID indicado
/usuarios POST Crea un usuario. Mínimo necesitas: username, password, token
/usuarios PUT Modifica un usuario. Mínimo necesitas: id_usuario, token
/usuarios DELETE Elimina un usuario. Mínimo necesitas: id_usuario, token
